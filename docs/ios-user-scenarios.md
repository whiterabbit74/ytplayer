# MusicPlay iOS — сценарии поведения пользователей (по текущему коду)

Ниже перечислены все пользовательские сценарии, которые доступны в iOS‑приложении по текущей структуре экранов и доступным действиям. Каждый сценарий описан как цепочка нажатий/действий и ожидаемая реакция.

## Легенда
- `SC-XXX` — идентификатор сценария.
- `Ожидание` — ожидаемая реакция/результат.

---

## 1. Запуск и авторизация

SC-001: Первый запуск без токенов
- Шаги: Запустить приложение.
- Ожидание: Открывается экран входа `LoginView`.

SC-002: Запуск с валидным токеном
- Шаги: Запустить приложение при наличии токена.
- Ожидание: Открывается основной экран с табами и мини‑плеером (если есть текущий трек).

SC-003: Ввод серверного URL
- Шаги: На `LoginView` ввести URL сервера.
- Ожидание: URL сохраняется для дальнейшей авторизации.

SC-004: Ввод email/пароля
- Шаги: На `LoginView` ввести email и пароль.
- Ожидание: Кнопка `Sign In` становится активной.

SC-005: Успешный вход
- Шаги: Нажать `Sign In`.
- Ожидание: Успешная авторизация, переход в `MainTabView`.

SC-006: Ошибка логина
- Шаги: Нажать `Sign In` с неверными данными.
- Ожидание: Показ текста ошибки на экране логина.

SC-007: Смена URL перед логином
- Шаги: Ввести новый URL, затем нажать `Sign In`.
- Ожидание: Запрос авторизации идет на новый URL.

---

## 2. Навигация между табами

SC-020: Переход в таб `Search`
- Шаги: Нажать таб `Search`.
- Ожидание: Открывается `SearchView`.

SC-021: Переход в таб `Playlists`
- Шаги: Нажать таб `Playlists`.
- Ожидание: Открывается `PlaylistsView`.

SC-022: Переход в таб `Favorites`
- Шаги: Нажать таб `Favorites`.
- Ожидание: Открывается `FavoritesView`.

SC-023: Переход в таб `Queue`
- Шаги: Нажать таб `Queue`.
- Ожидание: Открывается `QueueView`.

---

## 3. Поиск и результаты (`SearchView`)

SC-030: Ввод поискового запроса
- Шаги: Ввести текст в поле `Search`.
- Ожидание: По мере ввода появляются подсказки.

SC-031: Выбор подсказки
- Шаги: Нажать на подсказку.
- Ожидание: Выполняется поиск по подсказке.

SC-032: Поиск по Enter
- Шаги: Ввести текст и нажать Enter.
- Ожидание: Выполняется поиск и список результатов обновляется.

SC-033: Просмотр списка результатов
- Шаги: Пролистать список результатов.
- Ожидание: Отображаются треки с кнопками действий.

SC-034: Дозагрузка результатов
- Шаги: Нажать `Load More`.
- Ожидание: Подгружается следующая страница результатов.

SC-035: Воспроизведение трека из поиска
- Шаги: Нажать кнопку `play` у трека.
- Ожидание: Трек начинает играть, открывается полноэкранный плеер.

SC-036: Добавление трека из поиска в очередь
- Шаги: Нажать кнопку `add to queue` у трека.
- Ожидание: Трек добавляется в очередь воспроизведения.

SC-037: Добавление трека из поиска в избранное
- Шаги: Нажать кнопку `heart` у трека.
- Ожидание: Трек добавляется/удаляется из избранного.

SC-038: Добавление трека из поиска в плейлист
- Шаги: Открыть контекстное меню трека и выбрать плейлист.
- Ожидание: Трек добавляется в выбранный плейлист.

---

## 4. Плейлисты (`PlaylistsView`, `PlaylistDetailView`)

SC-040: Создание плейлиста
- Шаги: Ввести имя и нажать `+`.
- Ожидание: Плейлист создается, список обновляется.

SC-041: Удаление плейлиста
- Шаги: Свайп по плейлисту и удаление.
- Ожидание: Плейлист удаляется, список обновляется.

SC-042: Открытие плейлиста
- Шаги: Нажать на плейлист.
- Ожидание: Открывается `PlaylistDetailView` с треками плейлиста.

SC-043: Воспроизведение трека из плейлиста
- Шаги: Нажать `play` у трека.
- Ожидание: Трек играет, открывается полноэкранный плеер.

SC-044: Добавление трека из плейлиста в очередь
- Шаги: Нажать `add to queue` у трека.
- Ожидание: Трек добавляется в очередь.

SC-045: Добавление/удаление трека из избранного в плейлисте
- Шаги: Нажать `heart` у трека.
- Ожидание: Статус избранного меняется.

SC-046: Удаление трека из плейлиста
- Шаги: В `Edit` режиме удалить трек из списка.
- Ожидание: Трек удаляется из плейлиста.

SC-047: Перетаскивание треков в плейлисте
- Шаги: В `Edit` режиме перетащить трек.
- Ожидание: Порядок треков сохраняется.

---

## 5. Избранное (`FavoritesView`)

SC-050: Просмотр списка избранного
- Шаги: Открыть таб `Favorites`.
- Ожидание: Показан список избранных треков.

SC-051: Воспроизведение трека из избранного
- Шаги: Нажать `play` у трека.
- Ожидание: Трек играет, открывается полноэкранный плеер.

SC-052: Добавление трека из избранного в очередь
- Шаги: Нажать `add to queue` у трека.
- Ожидание: Трек добавляется в очередь.

SC-053: Удаление трека из избранного
- Шаги: Нажать `heart` у трека.
- Ожидание: Трек удаляется из избранного.

---

## 6. Очередь (`QueueView`)

SC-060: Просмотр очереди
- Шаги: Открыть таб `Queue`.
- Ожидание: Показан список треков очереди.

SC-061: Воспроизведение трека из очереди
- Шаги: Нажать `play` у трека.
- Ожидание: Воспроизведение начинается с выбранного индекса.

SC-062: Удаление трека из очереди
- Шаги: Свайп и удаление трека.
- Ожидание: Трек удаляется, индексы обновляются.

SC-063: Удаление трека через кнопку в строке
- Шаги: Нажать кнопку `add to queue` (в очереди это удаление).
- Ожидание: Трек удаляется из очереди.

SC-064: Перетаскивание треков в очереди
- Шаги: В `Edit` режиме перетащить трек.
- Ожидание: Порядок очереди меняется.

---

## 7. Мини‑плеер (`PlayerMiniView`)

SC-070: Появление мини‑плеера
- Шаги: Запустить воспроизведение любого трека.
- Ожидание: Внизу появляется мини‑плеер с информацией о треке.

SC-071: Открытие полноэкранного плеера
- Шаги: Нажать на мини‑плеер.
- Ожидание: Открывается `PlayerFullView`.

SC-072: Пауза/возобновление
- Шаги: Нажать кнопку `play/pause` в мини‑плеере.
- Ожидание: Воспроизведение ставится на паузу/возобновляется.

SC-073: Следующий трек
- Шаги: Нажать кнопку `next` в мини‑плеере.
- Ожидание: Воспроизводится следующий трек очереди.

---

## 8. Полноэкранный плеер (`PlayerFullView`)

SC-080: Закрытие плеера
- Шаги: Нажать `Close`.
- Ожидание: Плеер закрывается, остается основной экран.

SC-081: Просмотр информации о треке
- Шаги: Открыть плеер во время воспроизведения.
- Ожидание: Видны обложка, название, исполнитель.

SC-082: Перемотка
- Шаги: Передвинуть `Slider`.
- Ожидание: Происходит seek в аудио.

SC-083: Предыдущий/следующий трек
- Шаги: Нажать кнопки `prev` или `next`.
- Ожидание: Переключение на другой трек в очереди.

SC-084: Пауза/возобновление
- Шаги: Нажать центральную кнопку `play/pause`.
- Ожидание: Пауза/воспроизведение.

SC-085: Повтор текущего трека
- Шаги: Нажать кнопку `repeat.1`.
- Ожидание: Включается/выключается повтор одного трека.

SC-086: Избранное из плеера
- Шаги: Нажать `heart`.
- Ожидание: Трек добавляется/удаляется из избранного.

SC-087: Отображение состояния “нет трека”
- Шаги: Открыть плеер без активного трека.
- Ожидание: Текст “No track playing”.

## 9. Системные и фоновые сценарии

SC-090: Синхронизация состояния плеера
- Шаги: Проигрывание трека, ожидание периодической синхронизации.
- Ожидание: Состояние очереди и позиции синхронизируется с сервером.

SC-091: Синхронизация при уходе в фон
- Шаги: Перевести приложение в фон.
- Ожидание: Выполняется принудительная синхронизация состояния.

SC-092: Автопереход к следующему треку
- Шаги: Дождаться окончания текущего трека.
- Ожидание: Воспроизводится следующий трек.

SC-093: Повтор одного трека при завершении
- Шаги: Включить repeat и дождаться окончания трека.
- Ожидание: Трек начинается заново.

SC-094: Управление через системные медиа‑кнопки
- Шаги: Использовать кнопки play/pause/next/prev в Control Center/гарнитуре.
- Ожидание: Приложение корректно реагирует на команды.

## 10. Ошибки сети и обновление токена

SC-100: Ошибка запроса (поиск, плейлисты, избранное)
- Шаги: Выполнить запрос при отсутствии сети.
- Ожидание: Ошибка не приводит к падению, UI остается стабильным.

SC-101: Истекший access token
- Шаги: Сделать запрос с истекшим токеном.
- Ожидание: Происходит refresh, запрос повторяется.

SC-102: Ошибка refresh token
- Шаги: Refresh token невалиден.
- Ожидание: Запрос завершается ошибкой, требуется повторный вход.
