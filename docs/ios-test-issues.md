# MusicPlay iOS — найденные проблемы при проверке сценариев

Ниже перечислены проблемы, выявленные при проверке всех сценариев из `ios-user-scenarios.md` по текущему коду.

## BUG-001 — Некорректная обработка Server URL
**Сценарии:** SC-003, SC-007
**Симптом:** `AppState.updateBaseURL` принимает любую непустую строку и сохраняет ее в `appState.baseURL`, даже если URL невалиден. `APIClient.updateBaseURL` при этом не меняет базовый URL (если строка невалидна). В итоге:
- запросы идут на старый URL, а
- формирование URL для превью/обложек в UI использует невалидную строку (картинки не грузятся).
**Статус:** исправлено.

## BUG-002 — Play/Pause после перезапуска с восстановленным треком не запускает аудио
**Сценарии:** SC-072, SC-084
**Симптом:** Если после запуска приложения есть `currentTrack` (восстановленный из `PlayerSyncService`), кнопка play вызывает `PlayerService.resume()`, но `AVPlayer` не содержит `currentItem`. Воспроизведение не стартует.
**Статус:** исправлено.

## BUG-003 — Системные команды Next/Prev не переключают трек фактически
**Сценарии:** SC-094
**Симптом:** `MPRemoteCommandCenter` вызывает только `playerStore.playNext()`/`playPrev()`, но `AVPlayer` не получает новый трек. Состояние UI может поменяться, но аудио не переключается.
**Статус:** исправлено.

## BUG-004 — Перетаскивание очереди не обновляет текущий индекс
**Сценарии:** SC-064, SC-083
**Симптом:** В `QueueView` перетаскивание меняет порядок массива, но `PlayerStore.currentIndex` не пересчитывается. В итоге `next/prev` работают относительно старого индекса и переключают не тот трек.
**Статус:** исправлено.
